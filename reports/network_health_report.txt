1. DEFINITIONS AND FORMULATION
------------------------------
The Network Health Index (NHI) is an explainable engineering metric scoring WSN operational
performance between 0 (complete failure) and 100 (optimal health). It is computed
by normalizing four distinct metrics and applying operational weights:

  * Battery Score (35% Weight):
    - Formula: battery_score = battery_level
    - Range: 0 to 100%

  * Signal Score (25% Weight):
    - Formula: signal_score = (signal_strength + 90.0) / 40.0 * 100.0
    - Range: RSSI >= -50 dBm maps to 100; RSSI <= -90 dBm maps to 0

  * Latency Score (20% Weight):
    - Formula: latency_score = (1500.0 - latency_ms) / 1400.0 * 100.0
    - Range: latency <= 100 ms maps to 100; latency >= 1500 ms maps to 0

  * Packet Loss Score (20% Weight):
    - Formula: packet_loss_score = (10.0 - packet_loss_rate) * 10.0
    - Range: packet_loss_rate = 0% maps to 100; packet_loss_rate >= 10% maps to 0

  * Combined Weighted Equation:
    nhi_score = 0.35*battery_score + 0.25*signal_score + 0.20*latency_score + 0.20*packet_loss_score
    (All subscores and final NHI score are clamped strictly to [0.0, 100.0])

2. HEALTH STATUS CLASSIFICATIONS
--------------------------------
The final NHI score classifies nodes into five distinct categories for maintenance dashboard feeds:
  * 90.0 - 100.0 ➔ EXCELLENT
  * 75.0 - 89.9  ➔ GOOD
  * 60.0 - 74.9  ➔ WARNING
  * 40.0 - 59.9  ➔ CRITICAL
  * 0.0  - 39.9  ➔ FAILING
